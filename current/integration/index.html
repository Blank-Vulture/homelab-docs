    連携フロー（現在）¶

    最終更新: 2026-03-22

    Cloudflare、Proxmox（LXC）、各サービスがどのように連携して動作するかを示す。

    外部アクセス経路（CF Tunnel 経由）¶

    sequenceDiagram
    actor User as ユーザー
    participant CF as Cloudflare Edge<br/>(WAF + Access)
    participant Tunnel as CF Tunnel
    participant CFD as cloudflared<br/>LXC 102 (10.10.0.10)
    participant TR as traefik<br/>LXC 101 (10.10.0.101:443)
    participant IM as immich<br/>LXC 100 (10.10.0.100:2283)

    User->>CF: HTTPS photos.rusticolus.dev
    CF->>CF: WAF チェック（日本IP・Bot・AI Crawl）
    CF->>CF: CF Access 認証
    CF->>Tunnel: 認証済みリクエスト転送
    Tunnel->>CFD: outbound tunnel 経由
    CFD->>TR: HTTP(S) Host: photos.rusticolus.dev
    TR->>TR: ルートマッチ (immich-cf.yaml)
    TR->>IM: http://10.10.0.100:2283
    IM-->>TR: レスポンス
    TR-->>CFD: レスポンス
    CFD-->>Tunnel: レスポンス
    Tunnel-->>CF: レスポンス
    CF-->>User: HTTPS レスポンス

    CF Tunnel が安全な理由¶

    脅威 対策
    WAN ポート開放 cloudflared は outbound 接続のみ。inbound ポートゼロ
    不正アクセス Cloudflare Edge で WAF + CF Access 認証が先行
    Bot / AI クローラー WAF ルール 4件でブロック
    日本国外からのアクセス Allow Only Japan WAF ルールでブロック
    通信傍受 CF Edge ↔ cloudflared 間は TLS 1.3 暗号化トンネル

    traefik が2つのルートを持つ理由¶

    ルートファイル Host エントリポイント 用途
    immich-cf.yaml photos.rusticolus.dev :443 CF Tunnel 経由の外部アクセス
    immich.yaml alpine-traefik.tail404f3b.ts.net :2283 Tailscale 経由の内部管理アクセス

    外部と内部で Host ヘッダーが異なるため、エントリポイントを分離している。 内部アクセス用ポート（2283）は Tailscale ネットワーク内からのみ到達可能。

    管理スクリプト連携¶

    flowchart LR
    SOPS["secrets/\nproxmox-homelab.sops.yml\n(age + YubiKey)"]

    subgraph SESSION["セッション (~/.cache/homelab/session.env)"]
        AGENT["ssh-agent\n(SSH秘密鍵登録済み)"]
        ENVVARS["CF_TOKEN_OPS\nPROXMOX_VE_API_TOKEN\n..."]
    end

    SOPS -->|"load-secrets.sh\n(YubiKeyタッチ1回)"| SESSION
    SESSION -->|"SSH_AUTH_SOCK"| ANSIBLE["ansible.sh\n(全LXC管理)"]
    SESSION -->|"CF_TOKEN_OPS"| CF_SH["cloudflare.sh\n(API操作)"]
    SESSION -->|"PVE_TOKEN"| PVE_SH["pve.sh\n(PVE API)"]
    SESSION -->|"TF_TOKEN等"| TOFU["tofu.sh\n(OpenTofu)"]

    eval "$(./scripts/load-secrets.sh)" を一度実行するとセッション（8時間有効）が作成され、 以降は YubiKey なしで全スクリプトが動作する。
