    ローカルLLM推論環境¶

    最終更新: 2026-02-11（初版） 関連ADR: ADR-002

    概要¶

    Claude Codeのバックアップとして、ローカルLLM推論環境を構築する。 既存の遊休GPUを専用ワークステーションに集約し、Qwen 2.5 32Bクラスのモデルを動作させる。

    ハードウェア構成¶

    本体: HP Z8 G4 Workstation（中古BTO）¶

    項目 スペック
    CPU Xeon Gold 6134 × 2 (16C/32T, 3.2GHz, TDP 130W)
    メモリ 64GB DDR4 ECC RDIMM (16GB × 4)
    ストレージ NVMe M.2 256GB (OS) + NVMe 1TB (モデル保管)
    PSU 1450W (P/N: 851385-001) ※要確認
    PCIe x16 × 4本 (2CPU構成時, PCIe 3.0)
    ネットワーク Intel 1GbE × 2 (オンボード)
    接続VLAN VLAN200 (Lab)

    GPU構成¶

    GPU アーキテクチャ VRAM TDP CUDA SM
    RTX 3080 (Palit) #1 Ampere (SM 8.6) 10GB GDDR6X 320W 68 SM
    RTX 3080 (Palit) #2 Ampere (SM 8.6) 10GB GDDR6X 320W 68 SM
    RTX 2080 Ti (GALAX) Turing (SM 7.5) 11GB GDDR6 250W 68 SM
    合計 31GB 890W

    電力バジェット¶

    GPU合計:    890W (RTX 3080 ×2: 640W + RTX 2080 Ti: 250W)
CPU合計:    260W (130W × 2)
システム:    ~50W
────────────────
合計:      ~1,200W

    Z8 G4のBTO標準PSUは1125Wのため、単体では不足する。

    HP純正1450W PSU (P/N: 851385-001) に換装する。

    1450W PSU 1台で全給電
→ 負荷率: 82.8% ✅（理想的な80%台）

    BTO注文時にPCServer1へ在庫を問い合わせること。

    1450Wが入手できない場合、外部ATX電源でGPUに給電する。 マイニング環境で広く使われた実績のある構成。

    Z8 G4 内蔵PSU (1125W)          外部ATX電源 (1000W)
  └─ CPU ×2, メモリ, NVMe,       └─ GPU ×3 (PCIe 8ピン ×6本)
     マザー, ファン                   ├─ RTX 3080 #1 (320W)
     → 負荷: ~310W (27.6%)          ├─ RTX 3080 #2 (320W)
                                      └─ RTX 2080 Ti  (250W)
                                      → 負荷: ~890W (89.0%)
    必要パーツ 価格
    ATX電源 1000W (80+ Gold) ~¥15,000〜20,000
    デュアルPSU連動基板（ADD2PSU等） ~¥500〜1,000
    PCIe 8ピン延長ケーブル ~¥500〜1,000

    連動基板がメインPSUの24ピンATX信号を検知し、外部PSUを同時ON/OFFする。 ライザーケーブルとの併用でGPU・ケーブルが外部に露出する構成になるが、 ホームラボ環境では実用上問題ない。

    デュアルPSU時の注意

    両PSUは必ず同じコンセント（電源タップ）から給電すること。 異なる系統から取ると電位差でマザーボードを損傷するリスクがある。

    ソフトウェアスタック¶

    推論エンジン比較¶

    フレームワーク 異種GPU 並列方式 推定速度 (32B Q4) 採用
    ik_llama.cpp ✅ グラフ分割 (テンソル並列) 25-40 tok/s ⭐ 推奨
    llama.cpp / Ollama ✅ レイヤー分割 (パイプライン) 8-15 tok/s 簡単だが遅い
    vLLM ❌ 同種GPU必須 テンソル並列 N/A 使用不可
    ExLlamaV2 ❌ Flash Attn制限 テンソル並列 N/A 2080 Tiで不可

    推奨モデル¶

    モデル パラメータ 量子化 VRAMサイズ 用途
    Qwen 2.5 32B Instruct 32B Q4_K_M ~20GB メイン: エージェント型タスク、IaCメンテナンス
    Qwen 2.5 14B Instruct 14B Q4_K_M ~9GB サブ: 軽量タスク、複数同時起動可
    Qwen 2.5 Coder 14B 14B Q4_K_M ~9GB コード生成特化（必要時）

    Qwen 2.5（非Coder）を選択する理由

    Claude Codeのバックアップ用途では、.claude/のスキル定義を読み取り、 Ansible/OpenTofu/Docker Composeの操作を自律的に実行する必要がある。 Coderモデルよりも汎用モデルの方がエージェント的思考・ツール活用に優れる。

    セットアップ手順¶

    1. ik_llama.cpp ビルド¶

    # 異種GPU対応: Turing (SM 7.5) + Ampere (SM 8.6)
git clone https://github.com/ikawrakow/ik_llama.cpp
cd ik_llama.cpp
cmake -B build \
  -DGGML_CUDA=ON \
  -DCMAKE_CUDA_ARCHITECTURES="75;86"
cmake --build build -j$(nproc)

    2. モデルダウンロード¶

    # Hugging Faceからダウンロード
mkdir -p /opt/models
huggingface-cli download \
  Qwen/Qwen2.5-32B-Instruct-GGUF \
  qwen2.5-32b-instruct-q4_k_m.gguf \
  --local-dir /opt/models

    3. 推論サーバー起動¶

    ./build/bin/llama-server \
  --model /opt/models/qwen2.5-32b-instruct-q4_k_m.gguf \
  --split-mode graph \
  --tensor-split 10,10,11 \
  --n-gpu-layers 999 \
  --ctx-size 8192 \
  --host 0.0.0.0 \
  --port 8080

    --tensor-split 10,10,11 は各GPUのVRAM比率に基づく分割。

    4. Open WebUI（フロントエンド）¶

    # docker-compose.yml
services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    ports:
      - "3000:8080"
    environment:
      - OLLAMA_BASE_URL=http://host.docker.internal:8080
    volumes:
      - open-webui-data:/app/backend/data
volumes:
  open-webui-data:

    PCIe帯域と推論性能¶

    なぜPCIe x4でも問題ないか¶

    LLM推論（特にレイヤー分割方式）では、GPU間で転送するデータは各レイヤーの活性化テンソルのみ。

    接続方式 帯域幅 LLM推論への影響
    PCIe 3.0 x16 ~16 GB/s ベースライン
    PCIe 3.0 x8 ~8 GB/s ほぼ差なし
    PCIe 3.0 x4 ~4 GB/s 実用上問題なし
    10GbE (ネットワーク分散) ~1.25 GB/s 致命的に遅い
    NVLink ~900 GB/s 理想だが高価

    HP Z8 G4（2CPU構成）はPCIe 3.0 x16 × 4本のため、3枚全てx16接続が可能。

    PVE-1/PVE-2への影響¶

    GPU移設後、PVE-1/PVE-2はCPUオンリーのワーカーノードとなる。

    影響 対応
    GPU-PT VMが使用不可 ゲームサーバー（Pterodactyl Wings）はCPUのみで稼働可
    Ceph OSDは影響なし ストレージはNIC (82599ES) 経由で継続
    監視エージェントは影響なし node_exporter等はCPUで動作

    ゲームサーバーへの影響

    Pterodactylで管理するゲームサーバー（Minecraft, Valheim等）はCPU主体のため、 GPU移設による影響はない。GPU-PT VMでのゲーミングが必要な場合は、 Z8 G4からGPUをPVE-1/2に戻すか、別途GPUを調達する。
