    ADR-005: DMZ ネットワーク物理分離 — 専用スイッチ FW 直結構成¶

    ステータス¶

    承認

    コンテキスト¶

    背景¶

    DMZ セグメント（VLAN 100）は外部公開サービスを収容する最高リスクゾーンだが、現状のネットワーク接続は以下の問題を抱えている:

    1. コアスイッチ経由: PVE-DMZ-1 がコアスイッチ（CRS305-1G-4S+）のポート4に直結。DMZ フレームが内部 VLAN（10/20/21/30/90/200/240/250）のフレームと同一物理スイッチ上を通過する
    2. ポート枯渇: CRS305 の SFP+ 4ポートが全て使用中（FW, Access SW, Lab SW, DMZ）。PVE-DMZ-2 を追加する物理ポートが存在しない
    3. MikroTik 脆弱性リスク: RouterOS は 2025年だけで CVSS 9.4 級の脆弱性が7件報告されている（ADR-001）。コアスイッチの侵害が全 VLAN に波及する構造
    4. 設計原則との矛盾: Lab（CRS309）やユーザーアクセス（CRS328）には専用スイッチがあるのに、最も攻撃面の大きい DMZ だけがコアスイッチ直結

    現状のトポロジー¶

    Internet → ONU → VP2440 ─SFP+ 1─→ CRS305 (Core)
                                     ├─ Port 1: FW (VP2440)
                                     ├─ Port 2: CRS328 (Access)
                                     ├─ Port 3: CRS309 (Lab)
                                     └─ Port 4: PVE-DMZ-1 ← DMZ が直結

    VP2440 ポート構成の誤記修正¶

    docs/hardware.md に「4x 2.5GbE, 10G SFP+」と記載されていたが、実際の VP2440 仕様は以下の通り:

    ポート チップセット 実数
    2.5GbE RJ45 Intel I226-V 2
    10G SFP+ Intel X710-BM2 2

    現在の使用状況:

    ポート 用途 状態
    2.5GbE 1 WAN（ONU） 使用中
    SFP+ 1 コアスイッチ（10Gbps trunk） 使用中
    SFP+ 2 - 空き
    2.5GbE 2 - 空き

    SFP+ の空きポートが1つあるため、DMZ スイッチの FW 直結が物理的に実現可能である。

    脅威モデル¶

    DMZ のネットワーク接続方式に関する攻撃ベクトルを分析した:

    カテゴリ 攻撃ベクトル 現状（コアSW直結）のリスク DMZ SW FW直結時のリスク
    SW 脆弱性 RouterOS CVE でコアスイッチ侵害 全 VLAN のフレームが攻撃者に露出。DMZ ↔ 内部の L2 攻撃が可能 DMZ トラフィックはコアスイッチを通過しないため無関係
    VLAN ホッピング 二重タギング攻撃 コアスイッチ上で DMZ と内部 VLAN が同居しているため攻撃面が存在 物理的に異なるスイッチのため攻撃経路なし
    DMZ ノード侵害 PVE-DMZ-1 root 奪取後の横展開 コアスイッチの L2 テーブルを汚染し、内部 VLAN のフレームを傍受する可能性 DMZ スイッチの L2 テーブル汚染は DMZ 内に限定。FW を突破しない限り内部到達不可
    物理アクセス ラック内ケーブル差し替え DMZ ポートが内部 VLAN ポートと隣接（コアスイッチの Port 3 と Port 4） DMZ スイッチは物理的に独立。差し替えても DMZ 内に限定
    FW 侵害 OPNsense root 奪取 全セグメント到達可能 全セグメント到達可能（変化なし — FW は両構成で Single Point）

    検討した選択肢¶

    選択肢 説明 評価
    A. DMZ SW をコアスイッチ配下 CRS305 Port 4 に DMZ SW を接続 ポート拡張のみ。DMZ トラフィックは依然コアスイッチを通過。物理分離にならない
    B. DMZ SW を FW に直結（採用） VP2440 SFP+ 2 に DMZ SW を接続 真の物理分離。コアスイッチを一切通過しない。FW が直接 DMZ を収容
    C. FW の 2.5GbE ポートに直結 PVE-DMZ-1 を VP2440 2.5GbE 2 に直結 スイッチ不要だが 2.5Gbps 上限。PVE-DMZ-2 追加時にポートが足りない
    D. 現状維持 変更なし PVE-DMZ-2 を追加できない。セキュリティリスクが残る

    決定¶

    1. DMZ 専用スイッチを FW に直結¶

    VP2440 の SFP+ 2 に DMZ 専用スイッチを接続し、DMZ トラフィックをコアスイッチから完全に物理分離する。

    変更後のトポロジー:

    Internet → ONU
              │ 2.5GbE
         VP2440 (OPNsense)
          │ SFP+ 1          │ SFP+ 2
          ▼                  ▼
    CRS305 (Core)      CRS305 (DMZ) ← NEW
     │    │    │         │    │    │
  CRS328 CRS309 [空き]  DMZ-1 DMZ-2 [空き]
  Access  Lab           VLAN100

    2. ハードウェア: MikroTik CRS305-1G-4S+IN¶

    項目 値
    型番 CRS305-1G-4S+IN
    ポート 4x SFP+ 10G + 1x GbE（管理用）
    参考価格 ~¥2.5万円
    消費電力 最大12W（ファンレス）
    スイッチング容量 82Gbps（non-blocking）
    電源 DC ジャック x2（冗長）+ PoE-in（GbE）

    選定理由¶

    1. コアスイッチと同一機種: スペアパーツとして相互交換可能。運用知識も共有
    2. ポート数が最適: uplink 1 + PVE-DMZ-1 + PVE-DMZ-2 + 予備 1 = ちょうど4ポート
    3. ファンレス: ホームラボで静音運転
    4. SwOS モード運用: RouterOS ではなく SwOS（純粋 L2 スイッチ）で動作させ、ルーティング機能を無効化。RouterOS の CVE リスクを大幅に低減
    5. PoE 給電対応: GbE ポートで PoE-in 対応。将来的に CRS328 から給電すれば電源ケーブル不要

    他の候補を除外した理由¶

    候補 除外理由
    CSS610-8G-2S+IN SFP+ が2ポートのみ。uplink + PVE-DMZ-1 で埋まり、PVE-DMZ-2 を 10G 接続不可
    CRS309-1G-8S+IN 8ポートは DMZ に過剰。価格も約2倍（~¥4万円）
    他メーカー（TP-Link 等） MikroTik 統一環境に異なるベンダーを混入すると管理コスト増。SwOS/RouterOS での統一管理を維持

    3. ポート割り当て¶

    VP2440（変更後）¶

    ポート 用途
    2.5GbE 1 WAN（ONU）
    SFP+ 1 コアスイッチ（内部 VLAN trunk）
    SFP+ 2 DMZ スイッチ（VLAN 100 専用）
    2.5GbE 2 予備

    コアスイッチ CRS305（変更後 — ポート4が空く）¶

    ポート 用途
    1 FW（VP2440 SFP+ 1）
    2 アクセススイッチ（CRS328）
    3 ラボスイッチ（CRS309）
    4 空き（将来拡張用）

    DMZ スイッチ CRS305（新規）¶

    ポート 用途
    1 FW（VP2440 SFP+ 2）— uplink
    2 PVE-DMZ-1（10G）
    3 PVE-DMZ-2（10G、将来）
    4 予備
    GbE 管理アクセス（VLAN 90）

    4. SwOS モードの推奨¶

    DMZ スイッチは SwOS モードで運用する。

    項目 RouterOS SwOS
    L3 ルーティング あり なし
    ファイアウォール あり なし
    管理プロトコル SSH, API, WinBox, SNMP Web UI のみ
    攻撃面 広い（RouterOS CVE 対象） 最小（L2 スイッチング + Web UI）
    VLAN タギング あり あり

    DMZ スイッチに求められるのは VLAN 100 の L2 スイッチングのみ。RouterOS の高機能は不要であり、攻撃面を最小化するために SwOS が適切。

    5. OPNsense インターフェース設定¶

    VP2440 の SFP+ 2 を DMZ 専用インターフェースとして設定:

    項目 値
    インターフェース名 ix1（Intel X710 port 2）
    OPNsense 上の名前 DMZ
    IP アドレス 10.0.100.254/24
    VLAN タグ なし（アンタグド — DMZ スイッチ全ポートが VLAN 100）
    ファイアウォールルール 既存の VLAN 100 ルールをこのインターフェースに移行

    アンタグド運用の理由: DMZ スイッチには VLAN 100 のトラフィックしか流れないため、VLAN タギングは不要。設定を簡素化し、ミスの余地を減らす。

    結果¶

    • 物理分離: DMZ トラフィックがコアスイッチを一切通過しない。コアスイッチ侵害時の DMZ への影響をゼロにする
    • VLAN ホッピング耐性: DMZ と内部 VLAN が物理的に異なるスイッチ上にあるため、L2 攻撃経路が存在しない
    • PVE-DMZ-2 拡張: DMZ スイッチのポート3に 10G 接続可能。IP 10.0.100.2 は割り当て済み
    • コアスイッチのポート4解放: 将来のネットワークセグメント追加（例: 第2ラボスイッチ、監視専用セグメント）に使用可能
    • 攻撃面の最小化: SwOS モード運用により RouterOS の CVE リスクから DMZ スイッチを隔離
    • 運用一貫性: コアスイッチと同一機種（CRS305）のため、スペアパーツ共有・運用知識転用が可能
    • コスト: ~¥2.5万円（CRS305-1G-4S+IN）+ SFP+ DAC ケーブル ~¥3,000
    • hardware.md 修正: VP2440 のポート構成を実際の仕様（2x 2.5GbE + 2x 10G SFP+）に修正

    侵害シナリオ比較¶

    シナリオ 変更前（コアSW直結） 変更後（DMZ SW FW直結）
    コアスイッチ RouterOS 脆弱性 DMZ を含む全 VLAN のフレームが露出 DMZ は無関係
    DMZ ノードからの L2 横展開 コアスイッチ経由で内部 VLAN フレームに到達可能性 FW を突破しない限り内部到達不可
    DMZ スイッチ侵害 N/A DMZ セグメント内に限定
    FW 侵害 全セグメント到達可能 全セグメント到達可能（変化なし）
