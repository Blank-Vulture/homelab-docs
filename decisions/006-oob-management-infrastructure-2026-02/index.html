    ADR-006: アウトオブバンド管理基盤 — JetKVM / PXE / OOBスイッチ構成¶

    ステータス¶

    承認

    コンテキスト¶

    Proxmoxクラスタの障害・侵害時でも管理操作を継続できるアウトオブバンド（OOB）経路が必要。 以下の課題に対応する設計を決定する。

    課題1: BMCの統一管理 各ノードのベンダー固有BMC（HP iLO、Dell iDRACなど）は以下の問題を抱える。

    • ノードごとにファームウェア更新・CVE対応が異なる
    • iLOなど一部は商用ライセンスが必要
    • 攻撃面が広く、Proxmoxと独立して侵害されるリスクがある

    課題2: OOBスイッチのベンダー多様性 コアネットワークはMikroTik（RouterOS）で統一しているが、 OOBスイッチに同じMikroTikを使うと以下のリスクがある。

    • Proxmoxクラスタ侵害後、攻撃者がMikroTikのCVE・設定慣例の知識を流用して OOBスイッチまで横展開できる可能性がある
    • SwOSはRouterOSとは別実装だが、同一ベンダーエコシステムへの依存は排除できない

    課題3: PXEによる自動復旧 手動でISOインストールメディアを差し込む運用は障害対応を遅延させる。 常時稼働するPXEサーバーがあれば、JetKVMとの組み合わせでリモートから 完全な再インストール・再構成が可能になる。

    課題4: GPU監視の統一 PVE-GPU（HP Z8 G4）はNVIDIA GPU（RTX 3080 Ti + 3070 + 2080 Ti）を搭載。 将来的にAMD GPUを追加した場合、エクスポーターが異なる。 一貫した監視設計が必要。

    課題5: k3sクラスタのメモリ上限 Forgejo + forgejo-runner + 将来的なワークロード増加を想定すると、 Raspberry Pi 4（最大8GB）ではメモリが不足する可能性がある。

    決定¶

    OOB管理端末¶

    Lab・DMZ各ゾーンに物理管理端末を1台ずつ配置する。

    ゾーン 役割 接続
    Lab OOB Lab OOBスイッチ経由で全PVEノードに到達 VLAN 91
    DMZ OOB DMZ OOBスイッチ経由でDMZノードに到達 VLAN 101

    管理端末間にKVMスイッチは設置しない。 物理的に分離されたゾーンの管理端末を共用するとゾーン間の経路が生まれるため。

    OOBスイッチ: Yamaha SWX2310-10G × 2¶

    コアネットワーク（MikroTik RouterOS）と異なるベンダーを採用する。

    項目 決定
    製品 Yamaha SWX2310-10G
    台数 Lab × 1、DMZ × 1
    ポート構成 8× GbE + 2× SFP+ (10G)
    運用モード L2スイッチング専用、インターネット非接続
    採用理由 国産ベンダー、RouterOSとエコシステムが完全に異なる、現行生産品のため入手性が高い、中古SWX2200-8Gと価格差が小さい、10G SFP+アップリンク搭載

    MikroTik RouterOSの知識・CVEがOOBスイッチに流用不可となり、 Proxmoxクラスタからの横展開リスクを構造的に排除する。

    IP-KVM: JetKVM¶

    ゾーン 台数 対象ノード
    Lab 4台 PVE-1, PVE-2, PVE-3, PVE-GPU
    DMZ 1台〜（拡張可） PVE-DMZ-1（+ 将来 PVE-DMZ-2）
    • ベンダーBMC（HP iLO等）は無効化する。JetKVMに一本化することで管理面を統一。
    • JetKVMはHTTPS経由でIP-KVMアクセスを提供。PXEブートトリガーにも対応。

    JetKVM 拡張品構成¶

    電源制御用拡張品はノードの電源方式によって使い分ける。 JetKVM本体のRJ-12ポートに接続し、1台につき1個を装着する。

    ノード 電源方式 必要な拡張品
    PVE-1 ATX ATX Extension Board
    PVE-2 ATX ATX Extension Board
    PVE-3（M75q Gen5） DC barrel 90W DC Power Control Extension
    PVE-GPU（HP Z8 G4） ATX ATX Extension Board
    PVE-DMZ-1（M90q Gen5） DC barrel 135W DC Power Control Extension

    ATX Extension Board: 3台 / DC Power Control Extension: 2台

    Serial Console Extension は採用しない。 JetKVM本体のHDMIキャプチャでBIOS/UEFIレベルのアクセスを代替できる。 M75q Gen5・M90q Gen5はRS-232ポートを持たないため無効。 将来OPNsense（Protectli VP2440）へのJetKVM追加を検討する際に再評価する。

    DC Power Control Extension の主要仕様: - 対応電圧: DC 12〜20V（5.5×2.5mm バレルジャック、5.5×2.1mm アダプター同梱） - MCU: Raspberry Pi RP2040、電力監視: INA219センサー（電圧・電流・電力計測） - JetKVM本体への給電: RJ-12経由で同時給電可能（USB-C電源ケーブルが不要になる）

    JetKVM 調達先¶

    jetkvm.com の "Buy from Reseller" に掲載されている公式リセラー。 Mouser・DigiKeyには未取扱い。Amazon経由の第三者出品（BitPC等）は保証範囲が不明確なため非推奨。 日本国内のグレー市場品（¥15,000/台）は正規品の保証がないため非推奨。

    リセラー 拠点 本体価格 拡張品価格 備考
    iKOOLCORE 中国（Shenzhen） $89.99 要確認 全4種扱い、バルク対応、1年保証
    WisdPi 中国（Shenzhen） $89.00 要確認 全4種扱い、PoE Splitterも取扱い
    Welectron ドイツ（EU） €99.00 €24.90/個 Approved RPi Reseller兼務、EU向け

    推奨: iKOOLCORE 本体・ATX Extension Board・DC Power Control Extensionを一括購入可能。 1年保証付き（保証後も低コスト修理対応）。Shenzhen発送のため日本へのリードタイムが短い。

    JetKVM 代替候補（入手困難な場合）¶

    製品 価格目安 備考
    NanoKVM Full（Sipeed） ≈$40（¥6,000）/ AliExpress RISC-V、安価、機能はJetKVMと同等
    DIY PiKVM（Pi Zero 2W + HAT） ≈$50-60（¥8,000）/ 台 OSS実績豊富、Pi Zero 2W既所有分を活用可

    PXEサーバー: Raspberry Pi 5/16GB（SC1113）× 2¶

    ゾーン ホスト 役割
    Lab OOB (VLAN 91) pxe-lab PXEブート、netboot.xyz、TFTP/DHCP
    DMZ OOB (VLAN 101) pxe-dmz PXEブート、netboot.xyz、TFTP/DHCP
    • Pi Zero 2W（既所有）から Pi 5/16GB（SC1113） に変更。 将来的なメモリ要件（k3sクラスタワークロード増加）に対応するため。
    • JetKVMとPXEを組み合わせることで、現地作業ゼロのフルリモート再インストールを実現。

    k3sクラスタ構成¶

    Forgejo（Gitサーバー）とforgejo-runnerを3ノードHAクラスタで運用する。

    ノード スペック 役割
    pi-1 Pi 5/16GB k3s CP + worker、forgejo-runner
    pi-2 Pi 5/16GB k3s CP + worker、forgejo-runner
    k3s-vm Proxmox VM（Lab VLAN 200） k3s CP + worker、forgejo-runner

    3ノードでetcd quorumを維持。Proxmox VM障害時も2台のPiでクラスタ継続。

    GPU監視エクスポーター¶

    GPU製造元 エクスポーター 取得方法
    NVIDIA dcgm_exporter NVIDIA DCGM経由、温度・利用率・ECC・クロック等を網羅
    AMD node_exporter hwmon / amdgpuカーネルドライバー経由

    HP Z8 G4（RTX 3080 Ti + 3070 + 2080 Ti）はdcgm_exporterを採用。 AMD GPUを将来追加した場合はnode_exporterのhwmonコレクターで対応する。

    OOBネットワークアドレス設計¶

    VLAN 名前 サブネット 用途
    91 Lab-OOB 10.0.91.0/24 Labゾーン帯域外管理
    101 DMZ-OOB 10.0.101.0/24 DMZゾーン帯域外管理

    Tailscale（VLAN 90 Mgmt）はコントロールプレーン（ADR-004参照）。 OOB VLANは物理KVMアクセス・PXE・JetKVM専用。相互に分離する。

    結果¶

    得られるもの¶

    • 障害耐性: Proxmoxクラスタが全滅しても、OOB端末 + JetKVM + PXEで完全リモート復旧が可能
    • ベンダー多様性: OOBスイッチをYamahaにすることで、MikroTik侵害がOOBに波及しない
    • BMC一本化: JetKVMに統一することでHP iLO等のBMC固有CVEリスクを排除
    • 自動復旧: PXE + netboot.xyzにより、ISOメディア作業ゼロでOSの再インストールが可能
    • 監視網羅: dcgm_exporterによりNVIDIA GPUのECC・温度・スロットリング等を詳細監視

    トレードオフ¶

    • OOBスイッチをYamahaに変更することで、MikroTikとの運用知識が分散する → OOBスイッチはL2スイッチング専用（設定が単純）なので運用負担は最小限
    • Pi 5/16GB × 2のコスト増（Pi Zero 2Wと比較して） → k3sクラスタのメモリ拡張コストと合わせて一回で解決できる
    • JetKVM本体5台 + 拡張品（ATX Extension Board × 3、DC Power Control Extension × 2）の初期コスト → HP iLOのライセンス費用・管理工数と比較して長期的にコスト効率が高い → 公式リセラー（iKOOLCORE等）経由なら国内グレー市場より安価で正規品保証あり → 入手困難な場合はNanoKVM Full（≈$40）またはDIY PiKVMで代替可能
