    ADR-008: FW冗長化（CARP）とVP6670への段階的マイグレーション¶

    ステータス¶

    承認（2026-03-07）

    コンテキスト¶

    Protectli VP2440 を初代FWとして調達し、単一障害点のない冗長構成を最初から見据えて設計する。 FW障害時のネットワーク全断・リモートワーク中の業務停止リスクをゼロにすることが目標。

    より高性能な Protectli VP6670（Intel i7-1255U, 10C/12T）をMASTERとして追加し、 将来の10Gbps WAN環境・Suricata IDS/IPSの処理負荷増大にも対応可能なCARPペアを構成する。

    VP6670 を CARP MASTER（主系）、VP2440 を CARP BACKUP（待機系）として運用する。 Phase B の配線設計は 10GbE WAN対応を前提とした最終形とし、ISP 10Gbps契約時の機材買い直しをゼロにする。

    ISP回線：MAP-E（IPv6プラス）とCARPの関係¶

    当環境の回線はIPv6プラス（MAP-E方式）であり、PPPoEではなくMAP-Eを使用している。

    MAP-Eはステートレスなため、CARPとの組み合わせは以下のように動作する：

    MAP-E CARPの動作原理:
  両FWが同じIPv6 WANプレフィックスを取得 (DHCPv6-PD)
      ↓
  IPv6プレフィックスからMAP-E IPv4アドレス・ポート範囲が決定論的に導出される
      ↓
  CARP VIP = 仮想IPv6アドレス → フェイルオーバー時もMAP-E導出結果が同一
      ↓
  MASTER/BACKUP切替後も同一IPv4/ポート範囲で通信継続

    OPNsenseのMAP-E + CARP構成はサポートされており、 WANインターフェースにCARPのVHIDを設定することで透過的なフェイルオーバーが可能。

    代替案と却下理由¶

    候補 価格 却下理由
    VP6650（i5-1235U） $749 コア数・クロック差が小さいが、将来のIDS/IPS負荷を考えるとi7が安全マージン大
    VP6630（i3-1215U） $589 6コア止まり。10Gbps WAN時のSuricata処理が不安
    VP2440×2（同型CARP） $399+$399 性能が現行と同等。主系をアップグレードする本来の目的を果たさない
    Qotom/Topton製品 不明 coreboot非対応。VP2440との実績互換性が不明

    FWインターフェース割り当て¶

    Phase A（VP2440単独・一時構成）：

    ポート VP2440（単独FW）
    RJ45-1 WAN（ONU RJ-45直結、一時）
    SFP+1 LAN → コアスイッチ（10GbE）
    SFP+2 DMZ → DMZスイッチ（10GbE）

    Phase B以降（CARP最終構成・10GbE WAN対応済み）：

    ポート VP6670（MASTER） VP2440（BACKUP）
    RJ45-1 予備 予備
    RJ45-2 pfsync専用（直結） pfsync専用（直結）
    SFP+1 LAN/DMZ → per-VP6670-SW（10GbE） LAN/DMZ → per-VP2440-SW（10GbE）
    SFP+2 WAN → WAN SW CRS305（10GbE） WAN → WAN SW CRS305（10GbE）

    Phase B で SFP+2 を WAN に固定することで、ISP 10Gbps 契約時も FW・スイッチの買い直しゼロ。 WAN SW（CRS305）の ONU 接続を RJ-45 → SFP+ ポートに差し替えるだけで 10GbE WAN が完成する（Phase D）。

    SFP+トランシーバー / DACケーブル選定¶

    両FWのSFP+ポートとコアスイッチ・DMZスイッチ間の接続にはDAC（Direct Attach Cable）を使用する。

    品名 価格/本 用途
    DAC-10G-1M（1m DAC） $11 ラック内・近距離接続（推奨）
    DAC-10G-3M（3m DAC） $15 やや距離がある場合
    SFP-10G-SR（光・MMF） $16 光ファイバー配線が必要な場合

    コアスイッチ（CRS305）・DMZスイッチ（CRS305）はIntel X710との相互接続実績あり。MSA準拠DACのためベンダー間互換性は問題なし。

    Phase C: VP6670 MASTER化（本番切替）¶

    物理配線は Phase B と完全に同一。ソフトウェア設定変更のみ。

    物理変更: なし（新規ケーブル・機材の購入不要）

設定変更のみ:
  VP6670: advskew = 0  （最高優先 → MASTERに昇格）
  VP2440: advskew = 100 （低優先  → BACKUPに降格）

→ 設定保存後、CARP自動フェイルオーバーで1秒以内に切替完了

    Phase C 追加調達コスト：$0

    VP6670 故障で VP2440 N150（4コア）が MASTER に昇格した場合、 10Gbps 環境では Suricata のインライン処理が律速になる可能性がある。 緊急時 Runbook：VP2440 が長期 MASTER 時は Suricata を Tap/Monitor モードに切り替えてスループットを確保する。

    per-FW SW→DMZ SW接続 DAC-10G-1M × 2（各 per-FW SW SFP+3）
    FW→WAN SW接続 DAC-10G-1M × 2（各FWの SFP+2）
    CARP VHID WAN: VHID 1 / LAN各VLAN: VHID 10〜250（VLAN番号対応）
    CARP切替時間 1秒以内（pfsyncによるセッション状態同期）

    10GbE WAN対応（Phase D、ISP 10Gbps契約時）¶

    Phase B の配線設計で対応完了済み。ISP 側 ONT 交換時に以下の 1 作業のみ：

    • WAN SW（CRS305）の ONU 接続を RJ-45 → SFP+3 に差し替える
    • 新規スイッチ・FW・DAC の追加購入は不要（ケーブル/モジュール 1 本 $11〜$44 のみ）

    マイグレーション手順詳細¶

    Phase 0: VP6670の準備（オフライン、ダウンタイムゼロ）¶

    作業場所: 現行ネットワーク外
所要時間: 約2時間
    1. VP6670にOPNsenseをインストール（coreboot BIOSで起動）
    2. VP2440の設定をエクスポート（System > Configuration > Backups）
    3. VP6670にインポートし、基本設定を確認
    4. VP6670単体でのルーティング・NAT動作テスト（別回線またはテスト環境）
    5. MAP-E + CARP設定の事前検証

    Phase A→B: 最終アーキテクチャへの移行 + CARP構成投入（約90分、瞬断10秒以内）¶

    ダウンタイム: WAN SW 挿入時の瞬断（10秒以内）のみ
    1. WAN SW（CRS305 SwOS）と per-FW SW × 2（CRS305 SwOS）をラック/棚に設置
    2. per-VP2440-SW の VLAN 設定
    3. SFP+1（VP2440 上流）: trunk 全 VLAN
    4. SFP+2（→ コアSW）: trunk VLAN 100 を除く全 VLAN
    5. SFP+3（→ DMZ SW）: VLAN 100 のみ
    6. per-VP6670-SW も同様に VLAN 設定
    7. 瞬断ここのみ（10秒以内）: VP2440 SFP+1 を一時切断し per-VP2440-SW 経由に組み直す
    8. ⑤ VP2440 SFP+1 → per-VP2440-SW SFP+1
    9. ⑥ per-VP2440-SW SFP+2 → コアSW SFP+1
    10. ⑦ per-VP2440-SW SFP+3 → DMZ SW SFP+1
    11. VP2440 の WAN インターフェースを RJ45-1 から SFP+2 に変更（OPNsense 設定）
    12. WAN SW CRS305 RJ-45 ← ONU ①
    13. WAN SW SFP+1 → VP2440 SFP+2 ② （DAC接続）
    14. VP6670 をラックに設置、以下をケーブル接続
    15. ③ WAN SW SFP+2 → VP6670 SFP+2
    16. ④ VP2440 RJ45-2 ↔ VP6670 RJ45-2（pfsync直結）
    17. ⑧ VP6670 SFP+1 → per-VP6670-SW SFP+1
    18. ⑨ per-VP6670-SW SFP+2 → コアSW SFP+2
    19. ⑩ per-VP6670-SW SFP+3 → DMZ SW SFP+2
    20. VP2440 で CARP の設定を追加
    21. HASync (pfsync): System > High Availability
    22. CARP VIP追加: 各インターフェース（WAN/LAN VLAN群/DMZ）→ Firewall > Virtual IPs
    23. VP2440 の advskew=0（MASTER）、VP6670 の advskew=100（BACKUP）
    24. VP6670 を BACKUP として同期（VP2440→VP6670 へ設定プッシュ）
    25. 動作確認：VP2440 が MASTER、VP6670 が BACKUP

    フェイルオーバーテスト（約30分）¶

    ダウンタイム: テスト時に1秒×数回

    # VP2440のCARPを一時無効化 → VP6670がMASTERに昇格することを確認
# OPNsense: Interfaces > CARP Settings > Temporarily Disable CARP
# 確認後: VP2440のCARPを再有効化 → VP2440がMASTERに戻ることを確認

    確認項目： - [ ] CARP切替後もインターネット通信継続（MAP-E維持） - [ ] pfsyncによる既存TCPセッションの維持 - [ ] 全VLANの疎通確認 - [ ] DMZサービスの継続稼働

    Phase B→C: VP6670をMASTERに昇格（5分、ダウンタイム1秒以内）¶

    # VP6670: advskew = 0  → MASTER
# VP2440: advskew = 100 → BACKUP
# 設定保存後、VP6670が自動的にMASTERに昇格

    または、VP2440のCARPを手動で一時無効化してVP6670をMASTERに昇格させ、 advskew調整後にVP2440のCARPを再有効化する。

    Phase C後: 安定稼働確認（1週間観察）¶

    • Grafana/Prometheusでの両FW監視
    • 週次でのCARPフェイルオーバーテスト実施

    結果¶

    得られるもの¶

    • FW冗長化: VP2440またはVP6670いずれかが故障しても1秒以内にフェイルオーバー
    • 性能向上: 主系がN150（4コア）→ i7-1255U（10コア/12スレッド）に強化
    • Suricata IDS/IPS・CAKE QoS処理の余裕が大幅増加
    • 将来の10Gbps WAN対応基盤
    • 計画メンテナンスの容易化: 片系をオフラインにしてもサービス継続
    • 段階的な出費: Phase A（$22）→ Phase B（+$369 + VP6670）→ Phase C（+$0）→ Phase D（+$11〜$44）と分散
    • 将来の10GbE WAN: Phase B 設計で対応済み。ISP 変更時は WAN SW のケーブル 1 本差し替えのみ
