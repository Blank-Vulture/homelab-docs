    🔴 CRITICAL¶

    C-1 update-forgejo.yml — Python 正規表現の $ アンカー問題¶

    • ファイル: .forgejo/workflows/update-forgejo.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: シェルの python3 -c "..." 内で r'^v(\d+)\.\d+\.\d+$' の $ が shell 変数展開と干渉し、 Codeberg バージョン比較が常に失敗またはスキップされる可能性がある。 結果として major version 変更の警告が毎回出るか、バージョン判定が完全に無視される。
    • 修正方針: $ の前にバックスラッシュを追加するか、Python を別ステップに分離する。 または re.fullmatch() を使用して $ アンカーを不要にする。

    🟠 HIGH¶

    H-1 update-packages.yml — ホスト更新失敗の隠蔽¶

    • ファイル: .forgejo/workflows/update-packages.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: 各 LXC ホストのパッケージ更新が失敗してもスクリプトは続行し、 ジョブは success を返す。週次で失敗が検知されずサイレントドリフトが発生する。
    • 修正方針: 各ホストの終了コードを FAILED_HOSTS 変数に蓄積し、 ジョブ末尾で exit 1 を返すようにする。

    H-2 update-pve.yml — ノード名ハードコード + リブートタイムアウト不足¶

    • ファイル: .forgejo/workflows/update-pve.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題 (a): リブート後の k3s Ready 確認で grep -q Ready がノード名 k3s-node をハードコード。 ノード名変更時にサイレント失敗。
    • 問題 (b): 待機ループが 24×20s=480s。カーネル更新後のリブートが長引くと偽失敗。
    • 修正方針:
    • (a) k3s kubectl get nodes --no-headers | grep -q Ready で任意ノード名に対応
    • (b) ループを 30 回 (600s) に拡張

    H-3 cloudflare-rotate.sh — 旧トークン削除前の新トークン検証なし¶

    • ファイル: scripts/cloudflare-rotate.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: 新トークン作成 → (検証なし) → 旧トークン削除 の順。 新トークンに権限がない場合、旧トークンを削除済みのためロールバック不可能。 Cloudflare 管理が完全に失われるリスク。
    • 修正方針: 旧トークン削除前に新トークンで /user API を呼び出して動作を確認する。 失敗時は新トークンを削除してエラー終了する。

    H-4 tofu.sh — HCL へのパスフレーズ直接展開¶

    • ファイル: scripts/tofu.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: passphrase = "$TOFU_STATE_PASSPHRASE" で HCL に直接変数展開。 パスフレーズに " や $ が含まれると HCL が壊れ、state が暗号化されないまま実行される恐れ。
    • 修正方針: HCL の passphrase を環境変数参照 (env("TOFU_STATE_PASSPHRASE")) に変更するか、 printf %s で適切にエスケープする。OpenTofu の env 参照構文を使うのが最もクリーン。

    H-5 git-push.sh — トークンの URL 展開で特殊文字破損¶

    • ファイル: scripts/git-push.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: https://user:${TOKEN}@host/... でトークンを URL に直接埋め込み。 トークンに @ や # が含まれると URL がパースエラーになる。
    • 修正方針: git credential-store またはヘッダー認証 (-H "Authorization: token ...") を使用する。 あるいは python3 -c "import urllib.parse; print(urllib.parse.quote(token))" で URL エンコードする。

    🟡 MEDIUM¶

    M-1 update-pi-runner.yml — バイナリ置換後の動作確認なし¶

    • ファイル: .forgejo/workflows/update-pi-runner.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: act_runner バイナリをダウンロード後、チェックサム検証なしに即置換。 破損したバイナリでもエラーなく上書きされる。
    • 修正方針: ダウンロード後に sha256sum で検証し、失敗時は既存バイナリを保持して終了。

    M-2 dependency-review.yml — バージョン比較が文字列一致¶

    • ファイル: .forgejo/workflows/dependency-review.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: v1.9.0 と v1.10.0 を文字列比較すると v1.9.0 > v1.10.0 になる。 マイナーバージョンが 2 桁以上になると更新を見逃す。
    • 修正方針: Python の packaging.version.Version (または手動 tuple 比較) で semver 比較する。

    M-3 weekly-update.yml — ハングした前回実行が次回を永久ブロック¶

    • ファイル: .forgejo/workflows/weekly-update.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: cancel-in-progress: false のため、前回実行がフリーズすると 次の週次実行がキューでブロックされ続ける。
    • 修正方針: cancel-in-progress: true に変更し、各サブワークフローに適切なタイムアウトを設定する。

    M-4 ansible.sh / provision-lxc.sh — エラー時の ssh-agent 残存¶

    • ファイル: scripts/ansible.sh, scripts/provision-lxc.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: eval "$(ssh-agent -s)" でエージェントを起動するが、trap がないため エラー終了時にバックグラウンドプロセスが残存する。
    • 修正方針: trap "kill \$SSH_AGENT_PID 2>/dev/null" EXIT INT TERM を追加する。

    M-5 update-pi-runner.yml — act_runner の再起動をランナー自身が実行¶

    • ファイル: .forgejo/workflows/update-pi-runner.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: ランナー自身が systemctl restart act_runner を実行すると、 実行中のワークフローが強制終了される。
    • 修正方針: systemd の ExecStop + RemainAfterExit を使うか、 ジョブ完了後にグレースフルリスタートするように変更 (sleep 5 && sudo systemctl restart act_runner & で ジョブが先にクリーンアップされてから再起動)。

    🔵 LOW¶

    L-1 backup-network-configs.sh — 上書き前のバックアップなし¶

    • ファイル: scripts/backup-network-configs.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: 前回の .rsc ファイルをバックアップなしに上書き。部分的な export があると消える。
    • 修正方針: 保存前に .bak にリネームし、成功時のみ正規ファイルに移動する。

    L-2 bootstrap.sh — 冪等性なし¶

    • ファイル: scripts/bootstrap.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: 再実行すると重複インストールや既存 hook の上書きが発生する。
    • 修正方針: brew list <pkg> || brew install <pkg> 形式に変更。pre-commit も既存確認を追加。

    L-3 update-packages.yml — Alpine での reboot-required が Debian 前提¶

    • ファイル: .forgejo/workflows/update-packages.yml
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: /var/run/reboot-required は Debian 固有。Alpine では常に "no" を返す。
    • 修正方針: Alpine では apk upgrade の出力でカーネルパッケージの更新を確認する。

    L-4 cloudflare.sh — ゾーン ID をリクエストごとに再取得¶

    • ファイル: scripts/cloudflare.sh
    • 状態: ✅ 対処済み (2026-03-26)
    • 問題: 複数コマンドを連続実行すると毎回 API でゾーン ID を取得する。 Cloudflare レート制限に近づく恐れがある。
    • 修正方針: ゾーン ID を環境変数または一時ファイルにキャッシュする。

    🔒 設計上意図的 (対処不要)¶

    I-1 load-secrets.sh — セッション TTL 8 時間、YubiKey なしで有効¶

    • ファイル: scripts/load-secrets.sh
    • 状態: 🔒 設計上意図的
    • 理由: CI/自動化ツール (Claude Code 等) がセッションファイルを使用するため意図的に長い TTL を設定。 ADR-010 (Vault + Teleport + Authentik) で将来的により secure な認可基盤に移行予定。
